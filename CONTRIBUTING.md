# 🤝 Contributing to AI Fairness Engine

First off, thank you for considering contributing to AI Fairness Engine! 🚀  
We welcome contributions from developers, designers, researchers, and open-source enthusiasts.

---

# 📌 Project Goal

AI Fairness Engine aims to detect, visualize, and eliminate bias in automated AI decision systems such as hiring algorithms.

This project focuses on:
- Fair AI systems
- Explainable AI (XAI)
- Bias detection
- Ethical machine learning
- Transparent scoring systems

---

# 🚀 Getting Started

## 1️⃣ Fork the Repository

Click the **Fork** button on the top right of this repository.

---

## 2️⃣ Clone Your Fork

```bash
git clone https://github.com/your-username/ai-fairness-engine.git
cd ai-fairness-engine

3️⃣ Install Dependencies
Frontend
cd frontend
npm install
Backend
cd backend
npm install
4️⃣ Setup Environment Variables
Backend .env
GEMINI_API_KEY=your_api_key
PORT=5000
NODE_ENV=development
ALLOWED_ORIGINS=http://localhost:5173
Frontend .env
VITE_API_URL=http://localhost:5000
5️⃣ Run the Project
Backend
cd backend
npm run dev
Frontend
cd frontend
npm run dev

🧾 Branch Naming Convention

Use meaningful branch names:

feature/add-dark-mode
bugfix/fix-chart-overflow
docs/update-readme
✅ Commit Message Guidelines

Use clean and meaningful commit messages.

Examples:

feat: add fairness metric cards
fix: resolve mobile navbar overflow
docs: improve deployment guide
refactor: optimize scoring engine
🔥 Pull Request Process
Fork the repository
Create a new branch
Make your changes
Commit your changes
Push to your fork

📋 Pull Request Rules

Please ensure:

Code is clean and readable
No unnecessary files are added
UI changes include screenshots
Responsive design is maintained
Existing functionality is not broken
🐛 Reporting Bugs

When reporting bugs, include:

Bug description
Steps to reproduce
Expected behavior
Screenshots (if possible)
Browser/device information

This repository intentionally includes simulated biased scoring systems for educational and research demonstration purposes only.

These systems must NEVER be used in real-world hiring or decision-making systems.
